2 任务知识

2.1 信息熵

假设X=...X−2.X−1,X0,X1,X2...是有限字母表上的平稳随机过程。让P表示X的可能性分布，让EP表示对P的期望。那么X​的熵就是

H(X)≡H(P)≡−EPlogP(X0|X−1,X−2,...).(1)

如果对数的底为 2，则熵以比特为单位。可以证明 H(P) 也可以表示为

H(P)=limn→∞−EPlog⁡P(X0∣X−1,X−2,…,X−n)=limn→∞−1nEPlog⁡P(X1X2…Xn).(2)

如果这个过程是遍历的，那么 Shannon-McMillan-Breiman 定理（Algoet 和 Cover 1988）指出，几乎可以肯定

H(P)=limn→∞−1nlog⁡P(X1X2…Xn).(3)

因此，对于遍历过程，H(P) 的估计值可以从根据P随机抽取的足够长的样本上的 P 的知识获得。 当 P 未知时，H(P)的上界仍然可以从P的近似值中获得。假设平稳随机过程 M 是 P 的模型。由 M 测量的 P​的交叉熵定义为

H(P,M)≡−EPlog⁡M(X0∣X−1,X−2,…).(4)

在合适的规律性条件下，可以证明

H(P,M)=limn−EPlog⁡M(X0∣X−1,X−2,…,X−n)=limn→∞−1nEPlog⁡M(X1X2…Xn).(5)

如果 P是遍历的，那么可以证明对于P

H(P,M)=limn→∞−1nlog⁡M(X1X2…Xn).(6)

交叉熵 H(P,M) 与我们相关，因为它是熵 H(P)的上限。 也就是说，对于任何模型 M​，

H(P)≤H(P,M).(7)

H(P,M) 和 H(P)之间的差异是对模型M 不准确性的度量。更准确的模型会产生更好的熵上限。 结合等式（6）和（7），我们几乎可以肯定地看到 P​，

H(P)≤limn→∞−1nlog⁡M(X1X2…Xn).(8)

熵和交叉熵可以从文本压缩的角度来理解。众所周知，对于任何唯一可解码的编码方案（Cover 和 Thomas 1991），

EPl(X1X2…Xn)≥−Eplog⁡P(X1X2…Xn).(9)

其中 l(X1X2...Xn)是字符串 X1X2...Xn编码中的位数。结合等式 (2) 和 (9)，我们看到 H(P)是对从P提取的长文本字符串进行编码所需的每个符号的平均比特数的下限：

H(P)≤limn→∞1nEPl(X1X2…Xn).(10)

另一方面，使用模型M的算术编码方案（Bell、Clery 和 Witten 1990）将对序列 xlx2...Xn​ 编码为

lM(x1x2…xn)=⌈−log⁡M(x1x2…xn)+1⌉.(11)

比特，其中⌈r⌉表示不小于r的最小整数。 结合等式 (7) 和 (11)，我们看到H(P,M)是通过使用模型 M 对从 P提取的长文本字符串进行编码而实现的每个符号的位数：

H(P,M)=limn→∞1nlM(X1X2…Xn).(12)

将印刷英语视为 95 个可印刷 ASCII 字符的字母表的随机过程。 例如，这个字母表包括所有大小写字母、所有数字、空格、所有标点符号等。使用等式（8），可以估计英文字符熵的上限如下： 1. 构造一个有限字符串上的语言模型 M。2. 收集相当长的英文文本测试样本。3. 那么 H( English )≤−1nlog⁡M( test sample )(13)，其中 n是样本中的字符数。

2.2 语言模型

模型通过符号三元组频率来捕获英语的结构。 粗略地说，该模型通过将序列分解为标记和空间并计算相应标记序列的概率来估计字符序列的概率。然而因为对于固定的标记词汇，一些字符序列不会有任何这样的剖析，而另一些则有几个，情况会稍微复杂一些。 例如，序列 abc xyz 可能没有任何剖析，而序列bedrock可能被剖析为一个标记或两个标记而没有中间空间。通过引入一个可以解释任何拼写的未知标记来解决无法解析的序列的困难。最终的字符串由空格字符串分隔的大小写拼写组成。字符串的概率是字符串的联合概率和剖分的偶有部分之和：

M( charactersstring )=∑dissections M( charactersstring , dissection ).(14)

字符串和解剖的联合概率是四个因素的乘积：

M(characterstring,dissection)=Mtoken (tokens)Mspell (spellings∣tokens)Mcase (casedspellings∣spellings,tokens)Mspace (characterstring∣casedspellings,spellings,tokens).(15)

一个字符串的概率是

Mtoken (t1t2…tn)=Mtoken (t1t2)∏i=3nMtoken (ti∣ti−2ti−1).(16)

条件概率Mtoken(t3|t1t2) 被建模为四个估计量 fi 的加权平均值

Mtoken (t3∣t1t2)=λ3(t1t2)f3(t3∣t1t2)+λ2(t1t2)f2(t3∣t2)+λ1(t1t2)f1(t3)+λ0(t1t2)f0,(17)

其中权重 λi满足 ∑λi=1 和 λi≥0。

拼写模型生成一个拼写 s1s2...sk 给定一个标记。 对于除 unknown_token 和 sentence_boundary_token 之外的任何标记，模型都会生成标记的拼写。 对于 sentence_boundary_token，模型生成空字符串。 最后，对于unknown_token，模型首先根据泊松分布选择长度k，然后从可打印的ASCII字符中独立且均匀地选择k​个字符来生成字符串。 因此

Mspell(s1s2...sk|unknowntoken)=λkk!e−λpk,(18)

其中 λ 是训练文本中每个标记的平均字符数，1/p是可打印的 ASCII 字符数。

# 分句
def cut_sentences(content):
    end_flag = ['?', '!', '.', '？', '！', '。', '…', '......', '……', '\n']
    content_len = len(content)
    sentences = []
    tmp_char = ''
    for idx, char in enumerate(content):
        tmp_char += char
        if (idx + 1) == content_len:
            sentences.append(tmp_char)
            break
        if char in end_flag:
            next_idx = idx + 1
            if not content[next_idx] in end_flag:
                sentences.append(tmp_char)
                tmp_char = ''
    return sentences
# 判断是否是中文
def is_chinese(str):
    for i in str:
        if i >= '\u4e00' and i <= '\u9fa5':
            flag = True
        else:
            return False
    return flag

3.3 计算平均信息熵

最后就是对另一个文本进行词/字的平均信息熵计算，之前算好的词/字和双词/双字的概率存在字典中，通过调用算好的概率，用公式(6)来计算测试文本的值，这里也是先计算每一句的entropy，最后用所有句子的entropy总和除以句子总数。

def cal_entropy(path, v, biv):
    with open(path, 'r', encoding='ANSI') as file:
        sentences = cut_sentences(file.read())
        entropy_sum = 0
        for i in range(len(sentences)):
            entropy = 0
            sentences[i] = sentences[i].strip()
            t = jieba.lcut(sentences[i])
            flag = 1
            for j in range(len(t) - 1):
                if is_chinese(t[j]):
                    if flag == 1:
                        if t[j] in v:
                            entropy = v[t[j]]
                        else:
                            entropy = 1 / len(v)
                        flag = 0
                    for k in range(j + 1, len(t)):
                        if is_chinese(t[k]):
                            if t[j] + ' ' + t[k] in biv:
                                entropy *= biv[t[j] + ' ' + t[k]]
                            else:
                                entropy *= 1/len(biv)
                            j += 1
                            k += 1
                        else:
                            continue
                else:
                    j += 1
            if entropy >= 0.00000000000001e-128:
                entropy_sum += (-1) / len(t) * math.log(entropy, 2)
    return entropy_sum / len(sentences)
